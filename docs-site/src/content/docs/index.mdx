## Why it exists

When you work with a coding agent, context is expensive and it evaporates at the end of every conversation. You open a new chat and re-explain what was planned, what's done, what was decided, and what's stuck.

Pigeon gives you and the agent a shared workspace. You look at a kanban board. The agent reads and writes the same board through MCP tools. Nothing leaves your machine.

<CardGrid>
	<Card title="You see a board" icon="puzzle">
		Cards, columns, priorities, checklists, activity. Drag-and-drop in your browser. Everything the agent does shows up live.
	</Card>
	<Card title="The agent sees structure" icon="setting">
		Essential MCP tools are always on. Extended tools are discoverable on demand. The context footprint stays small.
	</Card>

	<Card title="Local-first by design" icon="laptop">

	</Card>
</CardGrid>
